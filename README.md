# UI-стресс: три подхода к нагрузке через браузер

Три популярных инструмента нагружают одно и то же веб-приложение через настоящий браузер: открывают страницу, заполняют форму, проверяют ответ в DOM.

---

## Выбранные инструменты

### 1. Grafana k6

k6 — современный open-source инструмент нагрузочного тестирования. Модуль `k6/browser` запускает Chromium и позволяет описывать браузерные сценарии на JavaScript.

### 2. Artillery + Playwright

Artillery задаёт профиль нагрузки в YAML (сколько пользователей, как быстро они появляются), а взаимодействие с браузером делегирует Playwright через внешний JS-модуль (processor). 

### 3. Selenium WebDriver + Python

Selenium - фреймворк для UI-автоматизации. Он не предназначен для нагрузочного тестирования напрямую, поэтому параллелизм создаётся вручную: Python-скрипт запускает несколько потоков, в каждом — свой экземпляр `webdriver.Chrome`. Каждый поток многократно проходит полный UI-цикл.

---

## Тестовое приложение

Простейший сервер на Flask (`app.py`, `dockerfile`):

- `GET /` — HTML-страница с формой: текстовое поле `username` и кнопка "Send Data".
- `POST /submit` — читает `username`, возвращает ту же страницу, но в блоке `#result` появляется текст `Saved: <имя>`.

В Docker Compose сервис `webapp` доступен тестам по адресу `http://webapp:5000`, с хоста — `http://127.0.0.1:5000`.

**Запуск**
```bash
docker compose up -d webapp
```

## Тесты

### Общий сценарий тестирования

**Сценарий**
1. Открыть `/`
2. Заполнить `username`
3. Кликнуть кнопку
4. Дождаться видимости `#result`
5. Проверить ответ и заголовок

### Selenium

| | |
|---|---|
| **Файлы** | `selenium_stress.py`, `Dockerfile.selenium` |
| **Образ** | собственная сборка (Python 3.12 + Chromium + chromedriver) |
| **Нагрузка** | 4 потока по 8 циклов = 32 полных UI-прохода |

Сценарий тестирования в случае с Selenium'ом включает некоторые дополнительные шаги.
Запускаются 4 потока, которые действуют по плану ниже.

**Сценарий одного потока:**
1. Создать headless Chrome
2. Начать цикл, повторить 8 раундов
   2.1 Открыть страницу (`driver.get`)
   2.2 Дождаться поля
   2.3 Очистить и ввести имя
   2.4 Нажать кнопку
   2.5 Дождаться `#result`
   2.6 Проверить текст и заголовок
3. После цикла закрыть браузер

**Запуск:**

```bash
docker compose run --rm selenium-test
```

Первый запуск требует сборки образа: `docker compose build selenium-test` (или автоматически, если образа ещё нет).

## Настройка нагрузки

В каждом тесте были заданы фиксированные параметры, однако при желании их можно изменять.
В случае с `Selenium` для этого даже были заведены переменные окружения.

- **k6:** `vus`, `duration` в `k6-script.js`
- **Artillery:** `phases` в `artillery.yml`
- **Selenium:** переменные окружения у сервиса или `docker compose run -e …`: `WEBAPP_URL`, `SELENIUM_WORKERS`, `SELENIUM_ROUNDS`

## Установка и запуск всех тестов

Нужны Docker и `docker compose`. Рабочая директория — каталог с `docker-compose.yml`.

```bash
docker compose up -d webapp
docker compose run --rm k6-test
docker compose run --rm artillery-test
docker compose run --rm selenium-test
docker compose down
```
